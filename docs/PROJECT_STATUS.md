## 主要风险或阻塞

| 风险 | 影响 | 当前状态 |
|---|---|---|
| `src/App.tsx` 过重 | 新逻辑默认堆入主入口，增加维护成本 | 已识别，本轮不拆代码，通过文档和规则限制增长 |
| 无自动化测试 | 功能回归全靠人工，改一处可能隐性破坏他处 | 本轮建立最小回归清单和测试策略文档，不搭建测试框架 |
| 轮询偏多 | Scheduler、页面同步使用轮询，有重复触发和性能风险 | 已识别，后续逐步替换为事件驱动 |
| 文档入口分散 | 新人或 AI agent 需要翻多个文件才能理解项目 | 正在通过本轮 PRD 解决 |
