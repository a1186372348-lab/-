# 开发检查清单

## 开发前检查

- [ ] 我已经写了功能需求卡，不是直接开始实现。
- [ ] 我知道这次改动影响哪些模块。
- [ ] 我知道状态真相在哪，不会随手复制一份状态。
- [ ] 我已经定义这次改动的完成标准。
- [ ] 我知道本次不做什么，避免范围失控。
- [ ] 我已经看过相关旧代码，不是盲改。

## 实现中检查

- [ ] UI 层只处理展示和交互，没有塞太多业务判断。
- [ ] 业务规则集中在 service 或明确的逻辑层。
- [ ] 新增的数据结构、字段、事件名称是统一的。
- [ ] 我没有为了省事把所有逻辑继续堆进入口文件。
- [ ] 我知道每个异步流程的开始、结束、失败点在哪里。
- [ ] 我给关键路径留下了足够的日志或可观察线索。

## 开发后检查

- [ ] `git status` 没有意外文件或构建垃圾。
- [ ] 至少做过一次构建检查或类型检查。
- [ ] 至少跑过一次主路径手测。
- [ ] 至少检查过一次异常路径。
- [ ] 至少检查过一次相关旧功能回归。
- [ ] 我能用自己的话解释这次改动为什么这样做。

## 最小回归清单

### 通用

- [ ] 应用能启动
- [ ] 主窗口正常显示
- [ ] 设置页可打开并保存

### 聊天与气泡

- [ ] 输入文本后能收到回复
- [ ] 回复能显示到气泡窗口
- [ ] 回复结束后气泡能正常关闭

### 待办

- [ ] 新增待办成功
- [ ] 修改待办成功
- [ ] 完成状态切换成功
- [ ] 删除待办成功

### 定时任务

- [ ] 可新增 daily 任务
- [ ] 可新增 interval 任务
- [ ] 触发后不会立即重复触发

### 专注模式

- [ ] 可以开始
- [ ] 可以暂停
- [ ] 可以重置
- [ ] 阶段切换时主窗口有联动

### 屏幕感知

- [ ] 未配置视觉 key 时不应报致命错误
- [ ] 配置后可正常走截图与分析流程
- [ ] 全屏或打扰模式时行为符合预期

## 发布前额外检查

- [ ] 文档与代码行为一致
- [ ] 没有把本地 key、本地脚本、本地缓存误提交
- [ ] 变更说明能让未来的自己看懂
