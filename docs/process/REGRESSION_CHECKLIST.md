# 最小回归清单

> 每次常规改动后，启动 `npm run tauri dev`，按以下清单逐项确认。
> 本清单覆盖最基本可观测行为，不替代需求卡中的完整验收清单。

---

## 使用方式

1. 先完成自动化检查：`npm run check`
2. 启动 `npm run tauri dev`
3. 从第 1 项开始逐项操作并记录结果
4. 全部通过 = 回归通过

---

## 1. 主窗口显示

| # | 验证操作 | 通过标准 |
|---|----------|----------|
| 1.1 | 观察桌面右下角 | 云朵渲染出现，无黑底或闪烁 |
| 1.2 | 观察云朵约 5 秒 | 云朵有随机眨眼或张望动画 |
| 1.3 | 按住 Ctrl 键 1 秒后松开 | 按住时云朵变为半透明且鼠标穿透；松开后恢复不透明且可交互 |

## 2. 子窗口打开

| # | 验证操作 | 通过标准 |
|---|----------|----------|
| 2.1 | 鼠标悬停云朵 | 悬停菜单出现，显示 4 个按钮（待办、设置、专注、定时任务） |
| 2.2 | 悬停待办按钮 200ms | 待办窗口在主窗口左侧弹出 |
| 2.3 | 悬停设置按钮 200ms | 设置窗口弹出 |
| 2.4 | 悬停专注按钮 200ms | 专注窗口弹出 |
| 2.5 | 悬停定时任务按钮 200ms | 定时任务窗口弹出 |
| 2.6 | 鼠标移出所有窗口区域 500ms | 当前打开的子窗口自动隐藏 |

## 3. 待办基本操作

| # | 验证操作 | 通过标准 |
|---|----------|----------|
| 3.1 | 在待办输入框输入"回归测试"，选择任意优先级，点击 "+" | 新待办出现在列表中，标题为"回归测试" |
| 3.2 | 点击该待办左侧复选框 | 待办标题显示删除线，播放完成动画 |
| 3.3 | 再次点击同一待办复选框 | 待办恢复未完成状态，删除线消失 |
| 3.4 | 鼠标悬停在该待办上，点击右侧 "×" 按钮 | 待办从列表中消失 |

## 4. 定时任务刷新

| # | 验证操作 | 通过标准 |
|---|----------|----------|
| 4.1 | 在定时任务输入框输入"回归提醒"，选择"间隔循环"模式，输入 1 分钟，点击 "+" | 新任务出现在列表中，显示"每 1 分钟" |
| 4.2 | 等待 1 分钟 | 云朵上方弹出语音气泡显示"回归提醒"，约 7 秒后气泡自动关闭 |
| 4.3 | 删除该定时任务 | 任务从列表中消失 |

## 5. 专注状态变化

| # | 验证操作 | 通过标准 |
|---|----------|----------|
| 5.1 | 打开专注窗口，点击计时器区域 | 倒计时开始，显示 MM:SS 格式递减 |
| 5.2 | 再次点击计时器区域 | 倒计时暂停，数字停止变化 |
| 5.3 | 再次点击计时器区域 | 倒计时继续递减 |
| 5.4 | 点击右上角重置按钮 | 计时器归零，回到初始状态 |

## 6. 语音气泡显示

| # | 验证操作 | 通过标准 |
|---|----------|----------|
| 6.1 | 在主窗口输入栏输入"你好"并发送 | 云朵切换为 thinking 表情，随后气泡在云朵上方淡入出现并显示 AI 回复 |
| 6.2 | 等待 AI 回复完成 | 回复内容在气泡中完整显示，约 5 秒后气泡自动淡出关闭 |
| 6.3 | 再次发送消息，在气泡显示期间点击气泡 | 气泡立即淡出关闭 |
