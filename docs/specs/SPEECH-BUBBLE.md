# 语音气泡窗口

> 状态：`shipped` | 最后更新：2026-04-17

---

## 1. 功能名称

语音气泡 — 在主窗口上方显示的浮动气泡，用于展示 AI 回复、提醒、问候等文字信息。

## 2. 目标

- **当前问题**：需要一个非侵入性的信息展示方式，让用户在不打开额外窗口的情况下看到 AI 回复和系统提醒
- **目标用户**：所有使用云朵助手的用户
- **期望结果**：气泡在云朵上方优雅出现，展示信息后自动消失，不干扰用户工作

## 3. 用户路径

### 3.1 AI 对话气泡

1. 用户在主窗口输入文字并发送
2. 主窗口发出 `speech:show` 事件（text = AI 回复首段，duration = 0）
3. 气泡在云朵上方淡入出现
4. 后续内容通过 `speech:append` 逐段追加
5. AI 回复结束后发出 `speech:done` 事件（duration = 5000）
6. 5 秒后气泡自动淡出关闭

### 3.2 提醒气泡

1. 定时任务/待办提醒触发
2. 主窗口发出 `speech:show` 事件（text = 提醒内容，duration = 7000）
3. 气泡淡入出现并停留 7 秒
4. 7 秒后自动淡出关闭

### 3.3 用户手动关闭

1. 气泡显示中
2. 用户点击气泡 → 立即淡出关闭

## 4. 输入 / 输出

### 输入

- **上游事件**：
  - `speech:show`：`{ text: string; duration?: number }` — 显示气泡，duration = 0 表示不自动关闭
  - `speech:append`：`{ delta: string }` — 追加文字（流式传输）
  - `speech:done`：`{ duration: number }` — 流式传输结束，启动关闭计时
  - `speech:hide`：无数据，立即隐藏
- **用户输入**：点击气泡关闭

### 输出

- **UI 变化**：气泡淡入/淡出动画，文字内容更新
- **数据写入**：无
- **外部调用**：无

## 5. 业务规则

- **自动关闭延迟**：
  - 普通消息：5000ms
  - 待办提醒 / 定时任务：7000ms
  - 专注结束：4000-5000ms
  - 流式传输：结束后 5000ms
- **duration = 0**：不启动自动关闭计时（流式传输期间）
- **点击关闭**：点击气泡立即关闭，清除所有计时器
- **鼠标悬停**：悬停时暂停自动关闭计时
- **位置跟随**：气泡位于主窗口正上方，跟随主窗口移动

## 6. 边界与异常

- **空文本**：`speech:show` 收到空 text 时仍显示气泡（气泡为空）
- **重复显示**：正在显示时收到新的 `speech:show`，新消息替换当前内容
- **事件乱序**：`speech:done` 在 `speech:show` 之前到达时忽略
- **窗口移动**：主窗口移动时气泡位置同步更新
- **流式中断**：流式传输中断（网络断开）后无 `speech:done`，气泡持续显示直到用户点击关闭
