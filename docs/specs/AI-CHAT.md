# AI 对话（AI Chat）

> 状态：`shipped` | 最后更新：2026-04-17

---

## 1. 功能名称

AI 对话 — 用户与云朵助手的核心交互能力，支持流式对话、上下文记忆、主动发言和屏幕感知。

## 2. 目标

- **当前问题**：用户需要一个能理解上下文、记住偏好的 AI 伙伴，而不是一次性的问答工具
- **目标用户**：使用云朵助手进行日常陪伴和轻量任务管理的个人用户
- **期望结果**：用户通过主窗口输入栏发送消息，AI 以云宝角色流式回复，回复展示在语音气泡中；AI 还能通过屏幕感知主动发起对话

## 3. 用户路径

### 3.1 用户发起对话

1. 用户悬停云朵 → 输入栏出现
2. 输入文字并按 Enter 提交
3. 云朵切换为 thinking 表情
4. 语音气泡打开，AI 回复以流式方式逐字显示
5. 回复完成后气泡 5 秒自动关闭
6. 云朵表情恢复

### 3.2 AI 主动发言

1. 屏幕监控服务每 30 秒截取一次屏幕
2. 通过视觉模型（Gemini/GLM）分析用户当前活动
3. AI 判断是否需要主动发言（基于时间段、活动内容、上次发言间隔）
4. 如果需要发言，云朵气泡自动弹出 AI 的话

### 3.3 对话记忆参与

1. 用户发送消息时，系统自动检索相关记忆和近期摘要
2. 记忆注入系统提示词，AI 基于个性化信息回复
3. 回复完成后自动从对话中提取新记忆

## 4. 输入 / 输出

### 输入

- **用户输入**：文字消息（通过主窗口输入栏）
- **上游事件**：无直接事件（由用户交互或屏幕监控触发）
- **依赖数据**：
  - `chat_history` 表：最近 20 条对话历史（超过 60 条自动压缩）
  - `user_memories` 表：相关记忆（通过向量相似度检索，上限 15 条）
  - `daily_summaries` 表：近 2 天的对话摘要
  - `settings` 表：`deepseek_api_key`（AI 服务密钥）、`vision_api_key`（视觉/嵌入服务密钥）

### 输出

- **UI 变化**：云朵切换 thinking 表情、语音气泡流式显示、阶段切换提示
- **数据写入**：
  - `chat_history` 表：保存用户消息和 AI 回复
  - `user_memories` 表：对话后异步提取记忆
  - `daily_summaries` 表：新一天首次对话时自动生成昨日摘要
  - `scheduled_tasks` 表：对话中识别到定时任务意图时自动创建
- **外部调用**：
  - DeepSeek API（`deepseek-chat` 模型）：对话生成
  - Gemini API（`gemini-2.5-flash-lite` / `gemini-embedding-001`）：屏幕分析和语义嵌入
  - GLM API（备选）：屏幕分析
- **事件发出**：
  - `speech:show` / `speech:append` / `speech:done` → 语音气泡窗口

## 5. 业务规则

- **角色设定**：云宝是治愈系桌面云朵助手，自称"云宝"，语言风格轻盈温暖，回复限制 3-5 句
- **流式响应**：通过 DeepSeek API stream 模式实现，`max_tokens: 300`，`temperature: 0.7`
- **上下文构建**：系统提示词 + 近期摘要 + 相关记忆 + 最近 20 条历史消息
- **历史压缩**：对话超过 40 条时，取即将被清理的旧消息交给 AI 提取长期记忆
- **日摘要**：新一天首次对话时，如果昨日对话 ≥ 5 条，自动生成 2-3 句摘要
- **屏幕监控**：30 秒间隔截图，基于哈希去重避免重复分析，3 分钟节流防止频繁发言
- **屏幕习惯**：每 10 分钟自动归纳屏幕行为为记忆（`screen_habit` 类型）
- **低干扰保护**：全屏游戏模式（disturbMode=2）时不触发主动发言
- **用户输入保护**：用户正在输入栏打字时不触发主动发言
- **API Key 缺失**：未配置 DeepSeek Key 时显示提示文字，不崩溃

## 6. 边界与异常

- **空输入**：用户提交空文本时忽略，不触发 AI 调用
- **API 调用失败**：捕获异常，通过气泡显示"出了点小问题，稍后再试试吧"
- **空回复**：AI 返回空内容时抛出错误，走异常路径
- **历史连续同角色**：DeepSeek 要求 user/assistant 交替，连续同角色消息只保留最早的一条
- **记忆提取失败**：提取过程静默失败，不影响对话体验
- **压缩并发保护**：`compressionInProgress` 标志防止多个压缩任务并行
- **摘要并发保护**：`summaryInProgress` 标志防止多个摘要任务并行
- **视觉 API 不可用**：Gemini/GLM 调用失败时静默返回空字符串，不触发主动发言
- **CORS 问题**：Gemini 使用原生 REST API（key 放 query param），避免浏览器 CORS 限制
