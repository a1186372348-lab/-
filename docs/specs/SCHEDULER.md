# 定时任务（Scheduler）

> 状态：`shipped` | 最后更新：2026-04-17

---

## 1. 功能名称

定时任务 — 独立子窗口中的定时提醒管理功能，支持每天定时和间隔循环两种触发模式，到期后通过语音气泡通知用户。

## 2. 目标

- **当前问题**：用户需要在固定时间或固定间隔收到自定义提醒，例如"每小时喝水"、"每天 10 点站会"
- **目标用户**：需要时间驱动提醒的桌面助手用户
- **期望结果**：用户创建定时任务后，系统按配置的时间或间隔自动触发提醒气泡，无需手动操作

## 3. 用户路径

### 3.1 创建定时任务

1. 用户悬停云朵 → 悬停菜单出现
2. 点击定时任务按钮 → 定时任务窗口在主窗口左侧弹出
3. 在输入框中输入任务标题
4. 选择触发模式：
   - **每天定时（daily）**：设置具体时间（如 10:00）
   - **间隔循环（interval）**：设置间隔分钟数（如 60 分钟）
5. 点击 "+" 按钮 → 任务插入数据库，列表刷新

### 3.2 编辑定时任务

1. 点击任务标题文字 → 进入内联编辑模式
2. 修改标题文字
3. 按 Enter 保存，按 Esc 取消

### 3.3 删除定时任务

1. 鼠标悬停在任务上 → 右侧出现 "×" 删除按钮
2. 点击删除按钮 → 任务从数据库移除，列表刷新

### 3.4 任务触发

1. 后台轮询服务每 60 秒检查一次数据库
2. 符合条件的任务触发时：
   - 更新 `last_triggered_at` 时间戳
   - 显示语音气泡提醒（持续 7000ms）
   - 云朵切换为 happy 表情

## 4. 输入 / 输出

### 输入

- **用户输入**：任务标题、触发模式选择（daily / interval）、时间或间隔设置
- **上游事件**：
  - `scheduler:reload`：新任务写库后通知 SchedulerPage 刷新列表
- **依赖数据**：
  - `scheduled_tasks` 表：定时任务数据（id, title, trigger_mode, daily_time, interval_minutes, action, is_enabled, created_at, last_triggered_at）

### 输出

- **UI 变化**：任务列表渲染、内联编辑、触发模式描述文本（如 "每天 10:00"、"每 60 分钟"）
- **数据写入**：
  - `scheduled_tasks` 表：插入、更新标题、删除、更新 last_triggered_at
- **事件发出**：
  - 语音气泡：触发时通过 `speech:show` 显示提醒内容
- **外部调用**：无

## 5. 业务规则

- **触发模式**：
  - `daily`：匹配 `daily_time`（HH:MM），每天最多触发一次（通过 `last_triggered_at` 日期判断）
  - `interval`：距离 `last_triggered_at` 超过设定分钟数时触发
- **轮询间隔**：60 秒（`POLL_INTERVAL_MS = 60_000`），启动后延迟 5 秒首次执行
- **触发动作**：当前仅支持 `notify`（语音气泡通知）
- **启停控制**：`is_enabled` 字段控制任务是否参与轮询检查
- **互斥显示**：定时任务窗口与待办窗口互斥，同一时间只能显示一个
- **窗口尺寸**：306 × 459px，透明背景，始终置顶

## 6. 边界与异常

- **首次触发**：`last_triggered_at` 为 null 时，interval 模式立即触发，daily 模式在下一个匹配时间触发
- **轮询精度**：最晚延迟 60 秒触发，不保证精确到秒
- **应用关闭**：应用关闭期间的任务不会被补偿执行
- **跨日边界**：daily 模式通过日期比较确保每天最多触发一次，不受轮询时间点影响
- **时间格式**：`daily_time` 使用 HH:MM 24 小时格式字符串存储

## 10. 本次不做什么

- 本次不涉及 cron 表达式支持
- 本次不涉及多种触发动作（仅 notify）
- 本次不涉及系统级定时器（Windows Task Scheduler）
- 本次不涉及跨设备同步
