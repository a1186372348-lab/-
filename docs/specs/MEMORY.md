# 记忆系统（Memory）

> 状态：`shipped` | 最后更新：2026-04-17

---

## 1. 功能名称

记忆系统 — AI 助手的长期记忆能力，支持从对话和屏幕行为中自动提取、存储、检索和纠正用户相关信息。

## 2. 目标

- **当前问题**：AI 对话缺乏持久性，每次对话都是全新状态，无法记住用户的偏好、身份和习惯
- **目标用户**：使用云朵助手进行日常交互的个人用户
- **期望结果**：AI 能自动从对话中学习用户信息，在后续对话中利用记忆提供个性化回复，无需用户手动配置

## 3. 用户路径

### 3.1 记忆自动提取

1. 用户与 AI 对话（发送消息 → 收到回复）
2. 对话完成后，系统异步分析对话内容
3. AI 提取值得长期记住的信息（身份、偏好、习惯、事实、任务）
4. 通过置信度门控（≥ 0.7）过滤低质量记忆
5. 通过精确去重和语义去重（向量相似度 ≥ 0.90）避免重复存储
6. 记忆写入 `user_memories` 表

### 3.2 记忆参与对话

1. 用户发送新消息
2. 系统自动检索与当前消息相关的记忆（向量相似度 × 0.50 + 重要性 × 0.25 + 近期度 × 0.15 + 关键词 × 0.10）
3. 记忆注入系统提示词的"关于用户的记忆"部分
4. AI 基于记忆上下文生成个性化回复

### 3.3 记忆纠正

1. 用户在对话中表达纠正意图（"你记错了"、"我不叫 XX"）
2. 提取引擎识别 `correction_of` 字段
3. 旧记忆标记为 `superseded`，新记忆以 `confirmed_count=2` 写入

### 3.4 历史压缩转记忆

1. 对话历史超过 40 条时，取即将被清理的旧消息
2. AI 提取旧消息中值得记住的信息
3. 转化为长期记忆保存

### 3.5 屏幕习惯记忆

1. 屏幕监控服务每 10 分钟归纳一次屏幕行为
2. 检查是否已有相似记忆（前 8 字符匹配）
3. 不重复时以 `screen_habit` 类型写入记忆

## 4. 输入 / 输出

### 输入

- **用户输入**：无直接用户输入（记忆系统是后台服务，由对话和屏幕监控驱动）
- **上游数据源**：
  - 对话消息（用户文本 + AI 回复）：触发 `extractMemoriesAsync()`
  - 旧对话历史（即将被清理的消息）：触发 `compressHistoryAsync()`
  - 屏幕描述（视觉模型输出）：触发 `maybeStoreScreenMemory()`
- **依赖数据**：
  - `chat_history` 表：提供待提取的对话内容
  - `settings` 表：`vision_api_key`（用于生成嵌入向量）

### 输出

- **数据写入**：
  - `user_memories` 表：存储提取的记忆（含向量嵌入）
  - `daily_summaries` 表：每日对话摘要
- **数据更新**：
  - 已有记忆的 `confirmed_count` 递增（重复出现时）
  - 已有记忆标记为 `superseded`（被纠正时）
- **外部调用**：
  - DeepSeek API：记忆提取和日摘要生成
  - Gemini API（`gemini-embedding-001`）：生成语义嵌入向量
- **事件发出**：无（记忆系统不直接发出 Tauri 事件）

## 5. 业务规则

### 记忆类型（fact_type）

| 类型 | 说明 | 默认重要性 |
|---|---|---|
| `identity` | 身份信息（姓名、年龄等） | 5 |
| `preference` | 偏好（喜欢/不喜欢） | 3-4 |
| `habit` | 习惯行为 | 2 |
| `fact` | 一般事实 | 1-2 |
| `task` | 任务相关 | 3-4 |
| `screen_habit` | 屏幕使用习惯 | 2 |

### 提取规则

- **主语判断**："云宝/云朵"是 AI 自身，`subject_role = "assistant"`；用户本人 → `subject_role = "user"`
- **置信度门控**：confidence < 0.7 的提取结果不入库
- **精确去重**：相同 `content` 的记忆不重复写入，`confirmed_count` +1
- **语义去重**：向量相似度 ≥ 0.90 视为重复
- **纠正机制**：用户纠正时标记旧记忆为 `superseded`，写入新记忆 `confirmed_count=2`

### 检索规则

- **混合评分算法**：`score = vecSim × 0.50 + importance × 0.25 + recency × 0.15 + keyword × 0.10 + confirmBonus`
- **返回上限**：单次检索最多返回 15 条相关记忆
- **嵌入模型**：使用 `gemini-embedding-001`，L2 归一化后余弦相似度等于点积

### 历史压缩规则

- **触发条件**：对话中 `!compressionInProgress` 且待压缩消息 ≥ 10 条
- **摘要触发**：新一天首次对话时，昨日消息 ≥ 5 条则生成摘要
- **并发保护**：`compressionInProgress` 和 `summaryInProgress` 标志防止并行

## 6. 边界与异常

- **提取失败**：记忆提取过程静默失败，不影响对话体验
- **嵌入 API 不可用**：嵌入生成失败时仍存储记忆（不含向量），后续检索仅依赖关键词匹配
- **日摘要对话过少**：昨日消息 < 5 条时不生成摘要，避免低质量摘要
- **压缩对话过少**：待压缩消息 < 10 条时不触发压缩
- **JSON 解析失败**：AI 返回的记忆 JSON 格式异常时静默跳过
- **无记忆时对话正常**：首次使用无任何记忆时，AI 仍正常对话，不注入记忆部分
- **记忆内容过长**：提取时截取对话前 3000 字符避免超出 token 限制
