# 提醒服务（Reminder）

> 状态：`shipped` | 最后更新：2026-04-17

---

## 1. 功能名称

提醒服务 — 后台定时检查未完成待办并通过语音气泡和音效提醒用户，按优先级选择最高优先的待办进行提醒。

## 2. 目标

- **当前问题**：用户创建待办后可能遗忘，需要系统主动提醒未完成的任务
- **目标用户**：使用云朵助手管理待办的个人用户
- **期望结果**：系统按设定间隔自动检查未完成待办，通过语音气泡和雷声音效提醒用户，避免重要任务被遗忘

## 3. 用户路径

### 3.1 自动提醒流程

1. 后台提醒服务按配置间隔（默认 60 分钟）定期检查
2. 查询所有未完成的待办（`is_completed = false`）
3. 过滤掉仍在冷却期内的待办（距 `last_reminded_at` 未超过间隔时间）
4. 按优先级排序（high > medium > low），选择最高优先级待办
5. 更新该待办的 `last_reminded_at` 为当前时间
6. 触发 UI 反应：
   - 云朵表情切换为 `worried`（担忧）
   - 播放雷声音效
   - 语音气泡显示 "「待办标题」is still pending."
7. 3 秒后云朵表情恢复为 `default`

### 3.2 提醒间隔调整

1. 用户打开设置窗口
2. 调整提醒间隔滑块（30 ~ 120 分钟）
3. 保存设置 → 发出 `settings-changed` 事件
4. 提醒服务在下次轮询时读取新间隔值

## 4. 输入 / 输出

### 输入

- **用户输入**：设置中的提醒间隔（30-120 分钟）
- **上游事件**：
  - `settings-changed`：设置保存后通知提醒服务更新间隔
- **依赖数据**：
  - `todos` 表：读取未完成待办及其 `last_reminded_at`
  - `settings` 表：读取 `reminder_interval_min` 配置值
- **配置默认值**：提醒间隔默认 60 分钟

### 输出

- **UI 变化**：云朵表情切换（default → worried → default），语音气泡弹出
- **数据写入**：
  - `todos` 表：更新 `last_reminded_at` 时间戳
- **音效播放**：雷声音效（`/sounds/thunder.mp3`）
- **外部调用**：无

## 5. 业务规则

- **提醒间隔**：通过 `setTimeout` 递归调用实现，间隔值动态从 `settings` 读取（30-120 分钟）
- **冷却机制**：每个待办通过 `last_reminded_at` 记录上次提醒时间，间隔内不会重复提醒同一待办
- **优先级选择**：每次只提醒一个待办，选择当前可提醒中优先级最高的（high > medium > low）
- **新待办处理**：`last_reminded_at` 为 null 的待办立即符合提醒条件
- **完成状态排除**：已完成的待办不参与提醒检查

## 6. 边界与异常

- **无未完成待办**：检查时没有未完成待办则不触发提醒，无异常行为
- **全部在冷却期**：所有未完成待办都在冷却期内则跳过本次提醒
- **设置变更时序**：`settings-changed` 事件触发后，下次轮询才生效，当前轮询周期不受影响
- **服务启停**：`startReminderService()` 返回停止函数，在 App.tsx cleanup 中调用
- **首次启动**：`last_reminded_at` 为 null 的待办在首次检查时即可被提醒
