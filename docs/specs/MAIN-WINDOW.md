# 主窗口（云朵桌面伴侣）

> 状态：`shipped` | 最后更新：2026-04-17

---

## 1. 功能名称

主窗口 — 桌面云朵伴侣的基础交互入口，负责云朵显示、表情切换、子窗口管理和全局状态协调。

## 2. 目标

- **当前问题**：用户需要一个始终在线的桌面助手，在不干扰工作的前提下提供 AI 对话、待办管理、专注计时等能力
- **目标用户**：日常使用桌面电脑的个人用户
- **期望结果**：用户在桌面右下角看到一只可交互的云朵，悬停显示菜单，可快速触发各项功能

## 3. 用户路径

### 3.1 基础交互

1. 用户看到桌面右下角的云朵图标（始终置顶）
2. 鼠标悬停云朵 20ms → 显示悬停菜单（4 个按钮：待办、设置、专注、定时任务）
3. 悬停菜单按钮 200ms → 打开对应子窗口
4. 鼠标移出主窗口和子窗口区域 500ms → 子窗口自动隐藏

### 3.2 AI 对话

1. 用户悬停云朵 → 出现输入栏
2. 用户输入文字并提交
3. 云朵切换为 thinking 表情
4. AI 回复通过语音气泡流式展示
5. 回复完成后气泡 5 秒自动关闭

### 3.3 穿透模式

1. 用户按下 Ctrl 键 → 窗口变为半透明（opacity 0.3）且可穿透
2. 松开 Ctrl 键 → 恢复正常显示

## 4. 输入 / 输出

### 输入

- **用户输入**：鼠标悬停、菜单点击、文字输入、Ctrl 键
- **上游事件**：
  - `settings-changed`：设置保存后通知（来自 SettingsPage）
  - `all-todos-complete`：所有待办完成（来自 TodoPage）
  - `cc-event`：Claude Code Hooks 事件（来自 bridge_server）
  - `focus-phase-change`：专注阶段切换（来自 focus 窗口）
  - `scheduler:reload`：定时任务刷新（来自 scheduler）
- **依赖数据**：
  - 天气数据（驱动 rainy 表情）
  - 屏幕截图（驱动主动发言）
  - 时间周期（驱动问候语）
  - 全屏检测（驱动低干扰模式）

### 输出

- **UI 变化**：云朵表情切换、菜单显示/隐藏、透明度变化、子窗口位置同步
- **数据写入**：对话记录、记忆提取结果
- **外部调用**：DeepSeek API（AI 对话）、Gemini/GLM API（屏幕感知）
- **事件发出**：
  - `speech:show` / `speech:append` / `speech:done` → 语音气泡窗口
  - `focus-start` / `focus-pause` / `focus-reset` → 专注窗口

## 5. 业务规则

- **互斥显示**：待办窗口与定时任务窗口互斥，同一时间只能显示一个
- **低干扰模式**：
  - 模式 0：正常显示（opacity = 1）
  - 模式 1：半透明（opacity = 0.15）
  - 模式 2：完全透明 + 穿透（opacity = 0，pointerEvents = none）
  - 触发：全屏应用检测（每 500ms 轮询）
  - 悬停主窗口时临时退出低干扰模式
  - CC 活动时强制显示（opacity = 1）
- **表情状态**：default / happy / thinking / worried / sleepy / rainy / sadly / proudly
- **空闲检测**：30 分钟无操作切换为 sleepy 表情
- **子窗口位置**：跟随主窗口移动同步调整，待办/定时在左侧，设置在右侧，专注在上方

## 6. 边界与异常

- **输入为空**：AI 对话提交空文本时忽略
- **API 不可用**：AI 对话失败时不崩溃，通过气泡显示错误提示
- **全屏应用**：游戏或全屏应用时自动进入低干扰模式，不遮挡用户内容
- **多窗口焦点**：失焦时重置所有悬停状态，隐藏菜单和子窗口
- **窗口穿透**：Ctrl 按下时 pointerEvents = none，松开恢复，避免用户被误触困住
- **屏幕截图去重**：基于哈希值判断屏幕是否变化，避免重复发送给视觉模型
