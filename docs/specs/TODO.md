# 待办管理（Todo）

> 状态：`shipped` | 最后更新：2026-04-17

---

## 1. 功能名称

待办管理 — 独立子窗口中的待办事项管理功能，支持创建、编辑、完成、删除和排序待办项，并提供日历视图。

## 2. 目标

- **当前问题**：用户需要一个轻量的任务管理工具，与桌面助手深度融合，在不需要切换应用的情况下管理日常待办
- **目标用户**：使用云朵助手管理日常任务的个人用户
- **期望结果**：用户通过悬停菜单快速打开待办窗口，以优先级驱动的方式管理任务，所有待办完成时获得正向反馈

## 3. 用户路径

### 3.1 创建待办

1. 用户悬停云朵 → 悬停菜单出现
2. 点击待办按钮 → 待办窗口在主窗口左侧弹出
3. 在输入框中输入标题，选择优先级（高/中/低）
4. 点击 "+" 按钮或按 Enter → 待办插入数据库，列表刷新

### 3.2 编辑待办

1. 点击待办标题文字 → 进入内联编辑模式
2. 修改标题文字
3. 按 Enter 保存，按 Esc 取消

### 3.3 完成待办

1. 点击待办左侧复选框 → 切换完成状态
2. 已完成待办显示删除线和烟花动画
3. 如果所有待办全部完成 → 发出 `all-todos-complete` 事件

### 3.4 删除待办

1. 鼠标悬停在待办上 → 右侧出现 "×" 删除按钮
2. 点击删除按钮 → 待办从数据库移除，列表刷新

### 3.5 排序和日历

1. 点击底部排序按钮 → 切换优先级排序方向（高→低 / 低→高）
2. 点击右下角日期芯片 → 展开/收起日历
3. 日历中绿色圆点标记有待办的日期，悬停日期显示当日任务

## 4. 输入 / 输出

### 输入

- **用户输入**：标题文字、优先级选择、复选框点击、标题点击编辑、删除点击
- **上游事件**：
  - `todo-mouse-enter` / `todo-mouse-leave`：鼠标进出通知（用于窗口焦点管理）
- **依赖数据**：
  - `todos` 表：待办数据（id, title, priority, is_completed, created_at, completed_at, last_reminded_at）
  - `todo_history` 表：历史归档（按 date_key 分日存储）
  - AI 在主窗口创建的待办（通过每秒轮询数据库检测新记录）

### 输出

- **UI 变化**：待办列表渲染、完成动画（删除线 + 烟花）、排序变化、日历展开/收起
- **数据写入**：
  - `todos` 表：插入、更新标题、更新完成状态、删除
  - `todo_history` 表：每日凌晨 5:00 自动归档过期待办
- **事件发出**：
  - `all-todos-complete`：全部待办完成时通知主窗口
- **外部调用**：无

## 5. 业务规则

- **优先级体系**：`high`（红色 #ff6b6b）> `medium`（黄色 #ffd93d）> `low`（绿色 #6bcb77），默认为 `medium`
- **ID 生成**：使用 UUID
- **自动清理**：每天凌晨 5:00 将已完成的旧待办归档到 `todo_history` 表，启动时和跨过 5:00 时触发清理
- **数据同步**：每秒轮询数据库检查新记录（支持 AI 在主窗口创建待办的场景）
- **互斥显示**：待办窗口与定时任务窗口互斥，同一时间只能显示一个
- **窗口尺寸**：306 × 459px，zoom: 0.85，透明背景，始终置顶

## 6. 边界与异常

- **空标题**：输入为空时不允许创建
- **日首次归档**：跨过凌晨 5:00 时触发归档，避免应用长时间运行导致已完成待办堆积
- **AI 创建待办**：主窗口 AI 创建的待办通过轮询机制同步到 TodoPage，最多延迟 1 秒
- **全部完成后新增**：如果全部待办完成后又新建待办，云宝表情不会再次触发 proudly
- **日历跨月**：日历组件支持月份前后切换，数据从 `todo_history` 和 `todos` 表查询
