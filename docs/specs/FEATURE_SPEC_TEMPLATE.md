# 功能需求模板

每次开发前先复制这一份，写完再开始实现。目标是把“我要做点什么”变成“我要交付一个可验证的功能”。

---

## 1. 功能名称

一句话命名，不要含糊。

示例：

- 定时提醒任务编辑
- 待办完成后播放奖励动画

这次改动到底解决什么问题？

- 目标用户：

## 3. 用户路径

用户怎么触发这个功能？

模板：

1. 用户在什么位置操作
2. 输入什么内容
3. 系统应该如何响应
4. 最终用户看到什么结果

## 6. 业务规则

## 7. 边界与异常

至少回答这些问题：

- 输入为空怎么办？
- 重复操作怎么办？
- 外部服务失败怎么办？
- 数据不存在怎么办？
- 多窗口或多状态冲突怎么办？

## 8. 完成定义

只有这里全部满足，才叫“功能完成”。

- 主路径可用
- 异常路径有处理
- 数据正确落库或正确更新
- 不破坏已有功能
- 有最小验证记录

## 示例提醒

如果你写完后，仍然不能回答下面这些问题，说明需求还不够清楚：

- 为什么要做这个功能？
- 改动会影响哪里？
- 成功的标准是什么？
- 失败时用户会看到什么？
