# 专注模式（Focus）

> 状态：`shipped` | 最后更新：2026-04-17

---

## 1. 功能名称

专注模式 — 独立子窗口中的番茄钟计时器，支持专注/休息双阶段自动切换，与云朵表情联动提供陪伴感。

## 2. 目标

- **当前问题**：用户需要一个轻量的专注计时工具，帮助建立工作节奏，在专注和休息之间自动切换
- **目标用户**：使用云朵助手进行日常工作的个人用户
- **期望结果**：用户通过悬停菜单快速打开专注窗口，设置任务后开始专注计时，云朵同步显示番茄钟进度，阶段切换时给出语音提示

## 3. 用户路径

### 3.1 开始专注

1. 用户悬停云朵 → 悬停菜单出现
2. 点击专注按钮 → 专注窗口在主窗口上方弹出
3. 可选：输入本次专注任务描述
4. 可选：调整专注时长（默认 25 分钟，最大 180 分钟）和休息时长（默认 5 分钟，最大 60 分钟）
5. 点击计时器 → 开始倒计时，云朵显示番茄钟进度环

### 3.2 专注进行中

1. 计时器每秒更新显示（MM:SS 格式）
2. 云朵上的番茄钟 SVG 实时显示进度和剩余时间
3. 点击计时器 → 暂停倒计时
4. 再次点击 → 继续倒计时

### 3.3 阶段切换

1. 专注倒计时结束 → 自动切换到休息阶段
2. 云朵气泡提示"专注结束啦，休息一会儿吧"，表情切换为 happy
3. 休息倒计时结束 → 自动切换到下一轮专注阶段
4. 云朵气泡提示"休息结束啦，重新专注吧！"

### 3.4 重置

1. 点击右上角重置按钮 → 回到当前阶段的初始状态（计时归零，停止运行）

## 4. 输入 / 输出

### 输入

- **用户输入**：任务描述文字、时长调节、开始/暂停点击、重置点击
- **上游事件**：无（专注窗口不监听主窗口事件）
- **依赖数据**：无持久化数据，所有状态在组件内存中管理

### 输出

- **UI 变化**：计时器显示更新、阶段指示（专注/休息颜色区分）、进度环动画
- **数据写入**：无
- **事件发出**：
  - `focus-start` → 主窗口：计时开始 `{ phase, remainSecs, task }`
  - `focus-pause` → 主窗口：计时暂停 `{ phase, remainSecs }`
  - `focus-reset` → 主窗口：计时重置 `{ phase }`
  - `focus-tick` → 主窗口/当前窗口：每秒更新 `{ phase, remainSecs }`
  - `focus-phase-change` → 主窗口：阶段切换 `{ phase, remainSecs }`
  - `focus-mouse-enter` / `focus-mouse-leave`：鼠标进出通知
- **外部调用**：无

## 5. 业务规则

- **默认时长**：专注 25 分钟，休息 5 分钟
- **时长上限**：专注最长 180 分钟，休息最长 60 分钟
- **阶段类型**：`focus`（专注，红色 #C94040）和 `rest`（休息，绿色 #4A9060）
- **自动切换**：倒计时归零时自动进入下一阶段，不停止计时器
- **云朵联动**：主窗口通过 `focus-tick` 事件实时更新番茄钟 SVG，通过 `focus-phase-change` 切换气泡提示和表情
- **窗口尺寸**：240 × 320px，透明背景，始终置顶，无装饰
- **窗口位置**：主窗口正上方居中

## 6. 边界与异常

- **任务描述为空**：允许空任务描述，仅显示计时器
- **跨阶段不中断**：阶段自动切换时计时器不停止，用户需手动暂停
- **窗口关闭丢失数据**：专注数据不持久化，关闭窗口后计时状态丢失
- **云朵未显示番茄钟**：如果主窗口未注册 focus 事件监听，番茄钟不显示但不影响专注窗口自身功能
- **时长调节限制**：计时运行中调节时长不立即生效，需重置后生效
