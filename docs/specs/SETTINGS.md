# 设置窗口

> 状态：`shipped` | 最后更新：2026-04-17

---

## 1. 功能名称

设置窗口 — 云朵助手的配置管理界面，用于管理 API 密钥、视觉模型选择和提醒间隔。

## 2. 目标

- **当前问题**：用户需要一个界面来配置 AI 服务密钥、屏幕感知能力和提醒频率
- **目标用户**：所有使用云朵助手的用户
- **期望结果**：用户通过简洁的设置界面完成配置，保存后立即生效

## 3. 用户路径

### 3.1 打开设置

1. 用户悬停云朵 → 出现菜单
2. 悬停设置按钮（齿轮图标）200ms → 设置窗口从主窗口右侧滑出
3. 设置窗口与主窗口间隔 8px，垂直居中对齐

### 3.2 修改设置

1. 用户在设置窗口中修改 DeepSeek API Key / 视觉模型 / 视觉模型 API Key / 提醒间隔
2. 点击"保存"按钮
3. 按钮变为"✓ 已保存"，2 秒后恢复"保存"
4. 设置写入 SQLite 数据库
5. 发出 `settings-changed` 事件通知主窗口

### 3.3 关闭设置

1. 鼠标离开设置窗口 500ms → 自动隐藏
2. 设置窗口位置跟随主窗口移动

## 4. 输入 / 输出

### 输入

- **用户输入**：
  - DeepSeek API Key（字符串，密码输入）
  - 视觉模型选择（gemini / glm / disabled）
  - 视觉模型 API Key（字符串，密码输入，仅在启用视觉模型时显示）
  - 待办提醒间隔（30-120 分钟，步长 5 分钟，默认 60 分钟）
- **依赖数据**：SQLite `settings` 表（key-value 存储）

### 输出

- **数据写入**：`settings` 表，使用 `INSERT ... ON CONFLICT UPDATE` 确保更新
  - `deepseek_api_key`：DeepSeek API 密钥
  - `vision_provider`：视觉模型提供者（gemini / glm / disabled）
  - `vision_api_key`：视觉模型 API 密钥
  - `reminder_interval_min`：提醒间隔分钟数
- **事件发出**：`settings-changed` → 主窗口（无数据负载，主窗口自行拉取新值）
- **UI 变化**：保存按钮显示"✓ 已保存"反馈

## 5. 业务规则

- **条件渲染**：视觉模型 API Key 输入框仅在选择了视觉模型（非 "disabled"）时显示
- **占位符适配**：
  - Gemini 选择时：`'AIza...'`
  - GLM 选择时：`'zhipu-api-key...'`
- **无前端验证**：API Key 格式由服务端验证，前端不做格式检查
- **保存反馈**：点击保存后显示"✓ 已保存"，2 秒后恢复"保存"文字
- **数据格式**：所有值以字符串形式存储在数据库中，数值类型在读取时转换

## 6. 边界与异常

- **API Key 为空**：允许保存空 Key，但 AI 对话功能不可用，应在调用时处理错误
- **视觉模型切换**：从 Gemini 切到 GLM 时，旧的 API Key 仍保留在数据库但 UI 显示为对应模型的新占位符
- **保存失败**：数据库写入失败时，用户看到的仍是"✓ 已保存"（当前无错误处理）
- **并发编辑**：只有一个设置窗口实例，不存在并发问题
- **窗口移动**：主窗口移动时设置窗口同步跟随
