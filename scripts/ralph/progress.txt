## Codebase Patterns
- App.tsx 是薄协调层（~96行），页面组装 + 用户交互入口 + AI 表现协调
- 窗口编排逻辑在 useWindowOrchestration hook（830行）：子窗口 show/hide、光标轮询、低干扰、穿透、hover 交互
- 运行时逻辑在 useAppRuntime hook（345行）：服务生命周期、事件桥接、空闲计时
- 服务遵循 start/stop 模式，返回 cleanup 函数
- Zustand store 用于全局 UI 状态（expression, weather, showHoverMenu, isProcessing）
- 子窗口通过 WebviewWindow.getByLabel() 访问，通过 emitTo() 通信
- 跨窗口事件当前使用字符串硬编码 + @tauri-apps/api/event 直接 import
- FocusClockState 在 3 个文件中重复定义（types.ts / useAppRuntime.ts / CloudRenderer.tsx / CloudPet/index.tsx）
- focusClock 和 ccActive 当前在 App.tsx 的 useState 中，目标迁入 Zustand
- FocusPage 对同一事件同时调用 emit() + emitTo()，存在冗余双重发送
- callbacksRef 模式是连接 hook useEffect 与外部回调的标准做法
- _resetIdle 模块级变量桥接 useAppRuntime 与 useWindowOrchestration 的循环依赖
- Tauri 桌面 GUI 测试需手动验证，自动化工具无法操作原生 WebView 窗口
- 事件 listen 在 L1 后已从 useAppRuntime 移回 App.tsx（437-508 行区间）
- npm run check 脚本在当前 branch 不存在，用 npx tsc --noEmit 替代
- speech:hide 和 scheduler:reload 的发送端不在前端代码中（可能来自 MCP 或外部系统）
- cc-event 由 Rust bridge_server.rs 通过 app.emit() 全局广播
